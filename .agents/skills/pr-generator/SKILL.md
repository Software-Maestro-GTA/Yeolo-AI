# Pull Request Generator Rule (PR 생성 규칙)

본 프로젝트에서 작업이 완료되어 Pull Request(PR) 본문 작성이 필요할 때 사용하는 스킬 가이드라인입니다. 

에이전트는 이 규칙이 트리거되면, 반드시 사용자가 규정한 아래의 마크다운(MD) 템플릿을 기반으로 변경 사항과 체크리스트를 충실히 채워 PR 상세 내용을 작성해야 합니다.

---

## 1. Pull Request 본문 표준 템플릿

PR 생성 요청을 받으면 아래 구조를 기본 뼈대로 삼아 마크다운 내용을 완성하십시오.

```markdown
## 변경 사항
이 PR에서 무엇을 왜 변경했는지 요약해주세요.

## 관련 이슈
Closes #(이슈 번호)

## 변경 유형
- [ ] 버그 수정 (기존 동작을 바꾸지 않는 수정)
- [ ] 새 기능 (기존 동작을 바꾸지 않는 추가)
- [ ] 호환성 깨짐 (기존 동작이 바뀌는 변경)
- [ ] 문서 수정

## 체크리스트
- [ ] 코드 스타일 가이드를 따랐습니다
- [ ] 셀프 리뷰를 마쳤습니다
- [ ] 필요한 곳에 주석을 추가했습니다
- [ ] 문서를 업데이트했습니다
- [ ] 테스트를 추가/수정했고 통과합니다
```

---

## 2. 세부 작성 지침

### 1) 변경 사항 (Changes)
* 단순히 코드 레벨의 변경점만 나열하지 말고, **"무엇을(What) 왜(Why) 변경했는지"** 비즈니스 목적과 기술적 맥락을 함께 요약합니다.
* 변경된 주요 모듈, API, 스키마, 또는 테스트 파일의 목록을 개조식으로 일목요연하게 정리합니다.

### 2) 관련 이슈 (Issues)
* PR과 관련된 이슈 번호를 `Closes #이슈번호` 형태로 정확하게 명시하여 PR 머지 시 이슈가 자동 닫히도록 처리합니다.

### 3) 변경 유형 & 체크리스트 (Type & Checklist)
* 구현 내용과 실제 작업 수준에 맞춰 `[ ]` 안에 `x`를 입력하여 선택 상태(`[x]`)로 표시합니다.
* 예: 문서를 업데이트했거나 테스트 코드를 추가해 통과했다면 이에 해당하는 체크박스에 `x`를 마킹합니다.
