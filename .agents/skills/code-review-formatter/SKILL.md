# Code Review Formatter Skill (코드 리뷰 결과물 포맷 지침)

이 문서는 소스코드를 검증(테스트 및 정적 분석)하고 피드백을 전달할 때 사용하는 코드 리뷰 리포트 규격서입니다.
리포트를 작성할 때는 임의의 서술 방식을 피하고, 아래에 규정된 **4대 오류 분류 체계**와 **마크다운 템플릿**에 맞추어 작성해야 합니다.

---

## 4대 코드 오류 분류 체계 (Issue Categorization)

발견된 모든 문제점은 다음 4가지 카테고리로 엄격히 분류하여 기술합니다.

1. **`LINT` (스타일 및 정적 분석 오류)**:
   - Python/Ruff 린트 경고 및 에러.
   - 신규/수정 소스 파일 최상단에 `module-explain-formatter`에 규정된 파일 설명 헤더(Docstring)가 누락되거나 규격에 맞지 않는 경우.
2. **`TYPE` (타입 무결성 오류)**:
   - Pydantic v2 데이터 검증 실패, Python 정적 타입 힌트 오류 및 불일치.
3. **`TEST` (테스트 실패 오류)**:
   - pytest 단위/통합 테스트 실패.
   - 외부 API/LLM 연동 시 `pytest-mocking` 규칙을 위반하고 실제 네트워크 요청이 발생한 경우.
4. **`LOGIC` (비즈니스 및 아키텍처 오류)**:
   - 기획 요구사항 및 인수 기준(Acceptance Criteria) 누락.
   - UI 컴포넌트(필요시)가 디자인 명세서 및 글로벌 스타일 CSS 변수(HSL 테마 등)를 위반하여 일관성을 해치는 경우.

---

## 코드 리뷰 피드백 마크다운 템플릿 (Feedback Template)

코드 리뷰 피드백 리포트는 다음 템플릿 형식을 준수하여 작성해야 합니다. 이 결과는 `log.md`에 기록됩니다.

```markdown
# Code Review Report: [기능 ID / 명세서 ID]

제출된 코드 검증 결과 다음 영역에서 개선이 필요합니다. 아래 내용을 반영하여 즉시 코드를 리팩토링하십시오.

---

## 발견된 오류 사항 (Issues)

### 1. [카테고리: LINT/TYPE/TEST/LOGIC] - [간략한 오류 내용]
* **발생 파일 및 위치**: [파일명.py](./상대경로/파일명.py#L라인번호)
* **문제 현상**:
  - [에러 메시지나 문제가 되는 코드 라인을 구체적으로 설명합니다.]
* **해결 가이드**:
  - [어떻게 코드를 고쳐야 테스트를 통과할 수 있는지 해결 방안을 구체적인 예시 코드와 함께 제시합니다.]

### 2. [카테고리: ...] - [간략한 오류 내용]
* **발생 파일 및 위치**: ...
* ...

---

## 종합 검증 요약 (Verification Summary)
* **Ruff Linter 린트 검사**: [PASS / FAIL]
* **테스트 커버리지 (pytest)**: [실패한 테스트 개수 / 전체 테스트 개수] (예: 0/14 Failed)
```
