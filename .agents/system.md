# Yeolo-AI Harness System Rules (system.md)

모든 Yeolo-AI 개발 및 명세 에이전트는 실행 패키지나 역할에 관계없이 아래의 전역 행동 제약 조건을 무조건 준수해야 합니다.

## 1. 프로젝트 및 환경 보존 규칙
- **패키지 매니저 준수**: 반드시 `uv`를 패키지 관리 및 스크립트 실행 도구로 사용하며, 임의로 pip install이나 다른 환경 구축 도구를 혼용하지 않습니다. 패키지 의존성 추가가 필요할 경우 반드시 `uv add`를 사용해 `pyproject.toml`과 `uv.lock`을 일관되게 갱신합니다.
- **오케스트레이션 제어 파일 보호**: `.agents/` 디렉토리 하위의 설정 파일(`config.json`), 실행 훅 스크립트(`hooks/` 하위), 규칙 정의서(`skills/` 하위)를 임의로 훼손하거나 동의 없이 수정하지 않습니다.

## 2. 상태 무결성 및 태스크 보존 규칙
- **독점적 진행 원칙**: 한 번에 오직 하나의 태스크만 `"진행중"` 상태여야 합니다. 이전 태스크의 리뷰가 완전히 승인되어 `"완료"`로 갱신되기 전까지 다음 태스크로 건너뛰거나 동시에 여러 태스크의 파일들을 변경하지 않습니다.
- **포맷 임의 변형 금지**: `progress.md`를 업데이트할 때, 기존 파일의 필드 구조나 데이터 타입을 훼손하지 않습니다.

## 3. TDD 파이프라인 준수 규칙
- **선제적 테스트 코드 작성 (Red Phase)**: `tester.md`가 실패하는 테스트 코드를 생성 및 확인하기 전에 `coder.md`가 먼저 실제 구현 코드를 개발하는 행위를 금지합니다.
- **테스트 결과의 객체성 신뢰**: 에이전트 자체의 추론에 의존하여 통과 여부를 임의 판단하지 않고, `config.json`에 정의된 실제 테스트 러너 실행 명령어(pytest)의 exit code 결과(`0`이면 성공, 이외 실패)를 기반으로만 검증 결과를 기록합니다.
- **테스트 수정 금지**: `coder.md`는 테스트를 강제로 통과시키기 위해 `tester.md`가 작성한 테스트 코드 파일을 임의로 조작하거나 완화해서는 안 됩니다.

## 4. 로깅 투명성 규칙
- **에러 로그 상세화**: 빌드 에러나 테스트 실패가 발생했을 때 이를 숨기지 않습니다. 터미널 표준 에러 출력(stderr) 전체를 `log.md`에 가감 없이 정확히 기입하여 디버깅 피드백 컨텍스트로 제공합니다.
